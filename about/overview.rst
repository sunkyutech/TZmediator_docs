################
はじめに
################

近年では，クラウドアプリケーションをエンドユーザの近くで実行することで，サービスの品質を向上させるエッジコンピューティングが普及してきています．
エッジデバイスにおいては，OSすら信頼できない場合がありますが，クラウドアプリケーションを安全に実行するために，CPUが提供するTrusted Execution Environment (TEE) を利用することができます．
TEEは，OSを含むシステムの他の部分からメモリを隔離し，攻撃者がTEEで使用されるコードやデータにアクセスできないようにします．
TEEのメモリが暗号化される場合には，物理的な盗聴も防ぐことができます．

エッジデバイス向けのTEEであるArm TrustZoneは，2つのワールドを提供しており，セキュアワールドでTrusted Application (TA) が動作し，ノーマルワールドでClient Application (CA) が動作します．
クラウドアプリケーション全体をセキュアワールドで実行することも可能ですが，セキュアワールドは高い権限を持っているため，クラウドアプリケーションに脆弱性があるとシステム全体に影響が及ぶ恐れがあります．
そのため，TAとして実行されるコードを最小限に抑えて，CAと連携しながらクラウドアプリケーションを実行することが望ましいです．
しかし，CAとTAの協調実行にはTEE用に定義された専用のAPIを使用する必要があり，CAとTA間の柔軟な協調は容易ではありません．

そこでTZmediatorは，2つのワールドに分割されたクラウドアプリケーションに対して，POSIX APIを用いて協調実行を行うことを可能にします．
TZmediatorは保護する必要がある処理のみをセキュアワールドでTAとして実行し，それ以外の処理はノーマルワールドでCAとして実行します．
POSIX APIを介してワールド間のシームレスな通信を実現するために，TZmediatorはノーマルワールド内に各TAに対応するシャドウプロセスを作成します．
TAがPOSIX APIを呼び出す際には，その呼び出しをシャドウプロセスに転送します．
より強力な分離を必要とする場合は，TA内でWebAssemblyアプリケーションを実行し，拡張されたWebAssembly System Interface (WASI) を介してCAと通信できるようにします．

TZmediatorは，OP-TEEに実装されており，TAとCAにそれぞれ提供されるTZmライブラリを開発しました．
TZm-TAライブラリは，TAがCAと通信できるようにPOSIX APIを提供します．
呼び出されたPOSIX APIは軽量な遠隔手続き呼び出し (RPC) またはポーリングベースの方式を使用してシャドウプロセスに転送されます．
現在，パイプやソケットなどのメッセージパッシングAPIや共有メモリAPI，シグナルAPIをサポートしています．
TZm-CAライブラリは，CAにPOSIX APIのプロセス生成に用いる関数を提供し，TAをシームレスに生成してシャドウプロセスを作成します．
